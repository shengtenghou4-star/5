# M3.1：分层收缩政治风险模型

## 为什么换模型

M3 的 1,110 个历史留出快照表明：原始多期限概率几乎不会互相交叉，PAVA 只改动了万分之一级别；真正的问题是相似案例模型在 30、90、180、365 天四个期限都没有击败简单历史基础率。

离任是低基准率事件。30 天留出集中只有 2 个正例，在这种场景下，“找最像的 50 个案例投票”很容易被少数偶然邻居拉偏。M3.1 因此引入透明的 empirical-Bayes 分层收缩模型。

## 模型层次

模型从平滑后的同期全局离任率开始，根据预先声明的配置逐级更新：

1. 全球同期风险；
2. 同一国家；
3. 同一国家内的任期阶段；
4. 同一国家、任期阶段和可观测政府形态。

每一层都以父层概率作为先验。局部样本少时，预测自动退回大样本父层；局部样本多时，才允许数据显著调整风险。模型不会使用未来标签、未结算案例或预测日之后的特征。

## 预先声明的选择网格

只在 2010–2014 年验证期选择：

- 模板：全球、国家、任期、国家×任期、国家×任期×政府形态；
- 任期桶：180、365、730 天；
- 收缩强度：25、100、400 个伪样本。

共 31 个候选。选择标准首先是四期限等权 adjusted integrated Brier；平分时依次偏好原始 Brier 更低、模型更简单、收缩更强、任期桶更宽的方案。

所有设置锁定后，2015+ 留出集只评估一次。改变最终留出期结果不得改变被选中的配置，相关回归测试已经写入。

## 命令

```bash
python -m fencha.m3_1_select_cli \
  --validation-start 2010-01-01 \
  --validation-end 2015-01-01 \
  --holdout-start 2015-01-01 \
  --horizons 30,90,180,365
```

结果写入：

```text
data/processed/m3_1_hierarchical_selection.json
```

M3.1 只有在锁定留出集上击败同期限历史基础率时才算实质改进。验证期获胜而留出期失败，仍按失败记录。
