# M3.4：按真实离任事件去重，并给路径成绩加置信区间

## 这一步修什么

M3.3 的路径成绩虽然明显提升，但原统计口径仍有两个依赖性问题：

1. 同一次领导人离任，会在多个预测日期重复出现；
2. 同一次离任，还可能同时进入 90、180、365 天等多个期限的汇总。

因此“58 个离任观察”不等于 58 次相互独立的政治交接。直接把它们当成独立样本，会让准确率看起来比证据本身更稳定。

## M3.4 的两项改动

### 1. 训练集按离任事件平衡

路径模型不再让同一次交接贡献多个月度样本。每个真实离任事件、每个预测期限只保留一个代表快照：进入该期限后最早的快照，也就是最接近期限边界、最难预测的那个时点。

这避免了一次长期酝酿的交接被重复几十次，压过其他独立事件。

### 2. 评估集按离任事件聚类

评估时同样执行：

- 每个真实离任事件在每个期限只计一次；
- 总成绩先在同一事件内部平均，再让每个事件获得相同权重；
- 置信区间按事件整组重采样，而不是把同一事件的多个期限拆成独立样本；
- 另外报告有多少国家的 Brier 分数实际得到改善，防止成绩只由少数国家驱动。

## 对比模型

同一份历史数据、同一批事件上同时比较：

- `conditioned_snapshots`：M3.3，只剔除未离任样本，但保留同一事件的多个历史快照；
- `event_balanced`：M3.4，每个历史离任事件只贡献一个代表快照。

基础线是预测时点之前、已经结算的独立离任事件中，两类交接路径的平滑历史比例。

## 证据解释

M3.4 仍是探索性稳健性审计，不是新的未见数据。它回答的是：

> M3.3 的路径优势在去掉重复事件、按事件聚类计算不确定性以后，还剩多少？

只有当 Brier 和 log loss 的改善区间大部分位于零以上，并且改善不是集中在极少数国家时，81% 左右的原始准确率才值得继续相信。
