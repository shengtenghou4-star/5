# M3.3：只在真实离任样本中学习路径

## 为什么继续改

M3.2 已经把“会不会离任”和“通过哪条路径离任”交给不同模型，但路径模型的训练集仍混入大量根本没有离任的样本。

这会把两个问题重新搅在一起：

- 一个路径标签为假，可能是因为领导人留任；
- 也可能是领导人确实离任，但走了另一条路径。

路径模型真正应该学习的是第二种对比：**在已经发生离任的条件下，哪条路径更像当前案例。**

## 训练过滤

M3.3 对机制模型增加一层严格过滤：

- 该机制为真的案例一定保留；
- 该机制为假，但标签在期限结束前已经结算，说明领导人通过另一条路径离任，也保留；
- 直到期限边界才以假结算的案例，代表期限内没有离任，从路径训练集中剔除。

这个判断只使用已经结算的历史标签日期，不增加任何预测时可见的未来特征。目标案例本身和尚未结算的案例仍然不可进入训练。

因此路径模型估计的是：

```text
P(具体路径 | 在该期限内已经离任)
```

总离任概率仍由 M3.1 的分层收缩模型负责，竞争风险层再把总概率分配到互斥路径，保持：

- 跨期限离任概率单调；
- 每条路径累计概率不下降；
- 所有路径概率精确加总为总离任概率。

## 证据等级

M3.3 仍属于探索性改进，因为模型设计是在观察过 2015+ 结果后提出的。它可以证明工程逻辑和同一历史区间内的增量表现，但不能冒充全新的独立盲测。

下一轮确认性检验应使用完全后移的时间块，或者在取得更新后的真实数据后锁定一次评估。
