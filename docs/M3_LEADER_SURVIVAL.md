# M3：领导人政治生存曲线

M3 把原来的单一问题——“未来 180 天内是否离任”——扩展成一条可以随时间更新的政治生存曲线。

## 这一步解决什么

单一 180 天概率看不出风险集中在哪个时间段，也容易掩盖模型之间的矛盾。M3 同时预测 30、90、180、365 天离任概率，并强制满足：

```text
P(30天内离任) <= P(90天内离任) <= P(180天内离任) <= P(365天内离任)
```

如果独立模型给出相互打架的结果，系统使用等权 PAVA（pool-adjacent-violators algorithm）做最小平方调整，生成合法的累计风险曲线。原始概率和调整后概率会同时保留，便于审计。

## 数据构造

```bash
python -m fencha.m3_cli build
```

默认生成：

```text
data/processed/parlgov_leader_survival.jsonl
data/processed/parlgov_leader_survival.manifest.json
```

每一个领导人、每一个月度截止点会拆成 30、90、180、365 天四个严格可结算的历史问题。各期限分别执行右删失检查：观察窗口没有完整结束的负样本不会进入训练集。

可以修改期限：

```bash
python -m fencha.m3_cli build --horizons 14,30,90,180,365
```

## 历史重放

先从 JSONL 中选一个 `case_id`，然后站回当时，只使用该截止时点之前已经结算的案例生成曲线：

```bash
python -m fencha.m3_cli forecast \
  --target-case-id 'parlgov:DEU:Merkel:2017-01-01:30d'
```

输出包含：

- 每个期限的原始离任概率；
- 调整后的累计离任概率；
- 仍在任概率；
- 各时间区间新增的离任风险；
- 历史基础率、有效相似样本量和特征覆盖率；
- 截至最长预测期限的限制平均生存天数（restricted mean survival time）。

## 当前边界

ParlGov 只能可靠标记“领导人是否发生交接”，不能单凭这张表判断交接原因。M3 当前不会把普通选举轮替、党内逼宫、辞职、死亡、政变等原因强行猜成标签。

下一阶段会接入可审计的离任原因标注和竞争风险模型，使总离任概率可以拆成若干路径：选举失败、联盟瓦解、党内撤换、主动辞职、健康或死亡、司法清算、军事政变及其他原因。原因标签必须有来源和置信等级，不能由新闻关键词直接冒充事实。
