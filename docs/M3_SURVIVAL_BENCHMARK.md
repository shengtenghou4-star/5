# M3：政治生存曲线历史盲测

M3 基础版可以同时生成 30、90、180、365 天离任概率，并用 PAVA 修正期限之间的逻辑冲突。本阶段增加时间安全的历史盲测，回答两个更重要的问题：

1. 多期限预测本身是否优于同期历史基础率；
2. 强制曲线单调以后，概率质量究竟改善还是恶化。

## 运行

先构造多期限 ParlGov 案例：

```bash
python -m fencha.m3_cli build
```

再运行盲测：

```bash
python -m fencha.m3_benchmark_cli \
  --holdout-start 2015-01-01 \
  --horizons 30,90,180,365 \
  --minimum-training-cases 500 \
  --target-stride 3 \
  --numeric-scale iqr
```

默认结果：

```text
data/processed/m3_survival_benchmark.json
```

## 时间安全

- 每个预测截止点只使用在该时点之前已经结算的历史案例；
- 各期限分别维护训练历史；
- 较早的留出期案例只有在其标签已经结算后，才允许进入后续预测；
- 只有 30、90、180、365 天标签齐全的同一历史快照才进入曲线评分；
- 结果标签必须随期限单调，即较短期限已经离任时，较长期限不可能重新变成“未离任”。

## 报告指标

- 各期限的基础率、原始模拟模型、PAVA 修正后模型的 Brier、log loss 和校准误差；
- 原始曲线发生期限交叉的数量和比例；
- 平均交叉幅度与平均修正幅度；
- 各期限等权的 integrated Brier；
- PAVA 修正相对原始概率和基础率的 Brier skill；
- 平均限制生存期（截至最长预测期限）。

PAVA 只保证逻辑一致，并不保证预测分数更高。若修正后的 Brier 变差，结果仍应如实保留，而不是为了曲线好看而隐藏。
